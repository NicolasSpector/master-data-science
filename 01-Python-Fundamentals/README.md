# 🔢 Guess the Number!

> **Final Project — Module 01: Python Basics** · Master in Data Science & Big Data

A terminal-based number-guessing game built in Python as the final submission for the introductory Python module of a Master's programme. The project focuses on practising control structures, modular programming, and data persistence through a fun and interactive game.

---

## 📋 Overview

The player (or players) must guess a randomly generated secret number within a limited number of attempts. Hints are provided after each wrong guess ("the number is higher / lower"). Every game session is saved to an Excel file so that a full statistics dashboard can be consulted at any time.

---

## ✨ Key Features

| Feature | Details |
|---|---|
| **Main Menu** | Clean, interactive text-based menu |
| **Single-player Mode** | Guess a number generated by the computer |
| **Two-player Mode** | Player 1 sets the secret number (hidden input); Player 2 guesses |
| **Configurable Difficulty** | Choose attempts: Easy (20), Medium (12), Hard (5) |
| **Configurable Range** | Choose number range: Easy (1–500), Medium (1–1 000), Hard (1–5 000) |
| **Statistics Dashboard** | 5 visualisation options powered by Matplotlib & Plotly |
| **Data Persistence** | All sessions saved automatically to an Excel file via openpyxl |

## 🚀 Getting Started

### 1 · Clone the repository

```bash
git clone https://github.com/<your-username>/guess-the-number.git
cd guess-the-number
```

### 2 · Install dependencies

```bash
pip install -r requirements.txt
```

### 3 · Play!

```bash
python menu_principal.py
```

> **That's it!** The stats file (`TAREA.xlsx`) is already included in the repository and the scripts find it automatically — no configuration needed regardless of where you downloaded the folder.

## 🎮 How to Play

```
Welcome to the best guessing game!! Choose a mode:
  1. Single-player
  2. Two-player
  3. Statistics
  4. Exit
```

1. **Select a game mode** from the main menu.
2. **Choose your difficulty** — number of attempts (Easy / Medium / Hard).
3. **Choose your range** — how wide the number pool is (Easy / Medium / Hard).
4. **Guess the number!** The game will tell you if the secret number is *higher* or *lower* after each attempt.
5. After the game ends, **check the Statistics menu** to review your performance.

## 📝 Conclusions

This project was designed as a practical exercise to consolidate the core Python concepts covered in Module 01:

- **Control structures** — `while` loops for the game loop and input validation; `if / elif / else` for game logic and menu navigation.
- **Modular programming** — the code is split into four focused modules (`funciones_juego`, `utilidades`, `visualizaciones`, `menu_principal`), each with a single responsibility.
- **Data persistence** — game results are written to and read from an Excel workbook using `openpyxl` and `pandas`.
- **Data visualisation** — multiple chart types (pie, bar, interactive tables) are generated with Matplotlib and Plotly to analyse player statistics.

It is a simple yet complete example that shows how these building blocks come together to form a cohesive, usable application.
